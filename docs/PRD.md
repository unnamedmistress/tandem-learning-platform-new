# Product Requirements Document (PRD)

## Jobs-To-Be-Done
1. Allow users to practice working with AI on real-world problems.
2. Provide a personalized learning experience with AI personalities.
3. Facilitate user reflection and growth through an onboarding journey.

## Pivots
1. **AI Persona Adaptation:** Allow users to switch between different AI personalities for varied perspectives.
2. **Skill Token Enhancement:** Implement a reward system tied to skill token achievements.
3. **Dynamic Onboarding:** Customize onboarding sequences based on user choices.

## Acceptance Criteria
1. Users can complete onboarding with three distinct stages: problem, AI chat, and reflection.
2. Six practice classes are available, each focusing on different AI skill sets.
3. Four AI personalities are selectable with unique interaction styles.
4. Skill tokens are tracked and redeemed within the platform.
5. AI chat provides feedback that is contextually relevant to each problem presented.
6. Users have the option to review detailed reflections after each session.
7. Platform supports both web and mobile responsiveness.
8. Version control is maintained through Git branches.
9. Documentation is generated automatically for each new release.
10. User progress is saved and can be resumed at any time.
